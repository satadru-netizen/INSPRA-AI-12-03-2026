# Blueprint Phase Template

Standard phases for Inspra AI voice agent projects. Customize sub-tasks per client.

## Phase 1: Requirements & Knowledge Base
**Node**: Gather Core Data
- Document target audience (industries, job titles, locations)
- Identify key personas / decision makers
- Collect existing knowledge base materials (FAQs, scripts, docs)
- Define business rules and compliance requirements

## Phase 2: Data & List Building
**Node**: Build Prospect Lists (outbound) / Contact Database (inbound)
- Source from Sales Navigator / Lusha / ZoomInfo / client CRM
- Import existing client contacts
- Filter by target criteria
- Validate and cleanse data
- Upload to platform (GHL / CRM)

## Phase 3: Platform Setup
**Node**: Platform Configuration
- Calendar integration (Outlook / Google Calendar)
- Create platform account (GHL / client system)
- SMTP / email configuration
- Notification setup (email, SMS, Slack)
- Booking automation rules

## Phase 4: AI Voice Agent Development
**Node**: Build Agent
- Train on domain-specific terminology
- Configure voice agent (inbound/outbound)
- Calendar/booking integration
- Demo setup (phone number or web link)
- System prompt development

## Phase 5: Testing & QA
**Node**: Quality Assurance
- Internal team tests as prospect/patient/customer
- Demo to client stakeholders
- Contact/data review in platform
- Feedback collection sheet
- Iterate based on findings

## Phase 6: Deployment
**Node**: Go Live
- Pre-launch meeting with client
- **Decision Gate**: Client Approval (Yes/No)
- Deploy to production
- Transfer to client's system (CRM, phone system)
- Monitor first 48 hours

## Phase 7: Website Chatbot (Conditional)
**Node**: Chatbot Setup
- Conversation flow design
- Requirements gathering
- Website integration
- Connect to platform (GHL / CRM)

## Phase 8: Future Scope (Conditional)
**Node**: Future Expansion
- Separate service proposal
- Not included in current scope
- Document for future engagement

## Visual Style Guide (Figma)
- **Project Start / Ongoing Support**: Green oval
- **Discovery items**: Red/pink rectangle
- **Standard phase items**: Blue rectangle
- **Platform/config items**: Orange rectangle
- **Build/development items**: Orange rectangle
- **Future/chatbot items**: Purple rectangle
- **Decision gates**: Yellow diamond
- **Flow**: Top-to-bottom, phases as vertical sections, sub-tasks branch horizontally
